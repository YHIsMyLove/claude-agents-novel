# 小说项目初始化

为你的小说创作项目创建标准化的目录结构和管理模板。

## 使用方式

```
/novel-assistant:init [项目名称]
```

## 初始化流程

系统将引导你完成以下步骤：

### 1. 项目信息

- **项目名称**：小说标题（将创建项目文件夹）
- **题材类型**：选择预设模板
  - `xianxia` - 仙侠/玄幻（修炼体系、宗门、灵兽、法宝）
  - `urban` - 都市/现代（职场、商战、都市异能）
  - `scifi` - 科幻（未来世界、科技、星际）
  - `custom` - 自定义（空白模板）

### 2. 创建目录结构

```
项目名称/
├── docs/
│   ├── world/           # 世界观设定
│   ├── characters/       # 角色库
│   ├── items/           # 物品库
│   ├── scenes/          # 场景库
│   ├── timeline/         # 时间线
│   └── maps/            # 地图/地点关系
├── outline/
│   ├── main-plot.md     # 主线大纲
│   ├── side-plots/       # 支线大纲
│   └── chapters/         # 章节大纲
├── chapters/
│   └── vol-01/          # 第一卷
│       └── ch-001.md     # 第一章
├── reviews/             # 评价记录
├── logs/                # 创作日志
└── novel.md             # 项目元数据
```

### 3. 生成项目元数据

创建 `novel.md` 包含：
- 项目标题、作者
- 题材、类型、标签
- 创作进度统计
- 设定状态追踪

### 4. 复制题材模板

根据选择的题材，复制对应的世界观、角色、物品模板。

## 创作理念

> **先快速框架，后逐步填充**

作者很难一开始就想好整个小说的架构。系统创建基础框架后，你可以随时补充和修改设定。

**设定变更提醒：**
- 🔴 **核心设定**（已发布章节关联的）- 修改需警告影响范围
- 🟡 **柔性设定**（未启用/未硬编码的）- 可自由修改
- 🟢 **草稿设定**（仅大纲阶段的）- 随时修改无影响

---

## 后续操作

项目创建后，使用以下命令开始创作：

- `/novel-assistant:outline` - 编写和检查大纲
- `/novel-assistant:write` - 写作助手
- `/novel-assistant:review` - 评价打分
- `/novel-assistant:status` - 查看项目进度
