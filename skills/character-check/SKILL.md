# 角色一致性检查

确保角色在你的小说中保持一致和立体。

## 检查维度

### 🎭 角色信息完整性

每个角色应该有：

```
【基本信息】
- 姓名、年龄、性别
- 外貌特征（身高、体型、特殊标记）
- 出身背景

【性格特质】
- 核心性格（3-5个关键词）
- 优点和缺点
- 说话风格/口头禅

【能力设定】
- 力量等级/能力列表
- 擅长技能
- 能力限制

【人物关系】
- 与其他角色的关系
- 对各方的态度
- 关系变化
```

### ⚠️ 一致性检查点

#### 行为与性格一致

```
问题: 角色 A 做出了与其性格不符的行为
示例: 谨慎的角色突然冲动行事
检查: 这行为是否有足够的动机/成长铺垫？
```

#### 能力边界一致

```
问题: 角色使用了未定义的能力
示例: 角色突然使用了新技能
检查:
  - [ ] 这是能力成长，需要更新能力列表
  - [ ] 这是设定错误，需要修改描述
  - [ ] 这是有意隐瞒，可以保留（需要伏笔）
```

#### 关系逻辑一致

```
问题: 角色关系与之前描述矛盾
示例: A 视 B 为友，但之前描述为敌对
检查: 中间发生了关系变化的事件吗？
```

#### 知识范围一致

```
问题: 角色知道不应该知道的信息
示例: 新角色知道只有核心角色知道的秘密
检查: 信息传递渠道是否合理？
```

## 常见问题

**Q: 角色太多记不住怎么办？**

A:
- 按重要程度分级：主角/重要配角/一次性角色
- 主要角色需要详细设定，次要角色只需关键特征
- 使用 `docs/characters/INDEX.md` 创建关系图

**Q: 角色性格发生变化怎么办？**

A: 这是好事！角色成长需要：
- 在角色卡中记录"变化前/变化后"
- 确保变化有足够的铺垫和事件促发
- 记录变化的时间点和原因

**Q: 如何让角色更立体？**

A:
- 给每个角色一个秘密（读者和部分角色不知道的）
- 给每个角色一个矛盾（如：胆小但想成为英雄）
- 让角色有独特的说话方式
- 通过小动作展现性格（如：紧张时摸戒指）
