# 时间线验证

确保你小说的时间线自洽合理。

## 检查维度

### 📅 事件时间一致

```
检查点:
- [ ] 事件日期是否与已有时间线一致
- [ ] 事件发生的先后顺序是否合理
- [ ] 事件之间的时间间隔是否合理
- [ ] 事件持续时长是否合理
```

### ⏳ 时间流逝合理

```
检查点:
- [ ] 场景间移动时间是否符合距离
- [ ] 过程描述的时间是否合理
- [ ] "一夜之间"、"数日后"等描述是否准确
- [ ] 角色在不同事件间的年龄变化
```

### 🔗 因果关系逻辑

```
检查点:
- [ ] 因果事件的时间顺序是否正确
- [ ] 前情是否有足够的准备时间
- [ ] 前情发展是否有足够的酝酿
- [ ] 多线并行时的时间线是否对齐
```

## 时间线记录模板

```markdown
# 时间线 - [主线/支线名称]

## 时间参考
- **纪元**: [如：天元历 3547 年]
- **当前季节**: [如：深秋]

## 事件记录

| 日期 | 事件 | 涉及角色 | 地点 | 重要性 | 备注 |
|------|------|----------|------|--------|------|
| 1/1 | 林凡入青云宗 | 林凡 | 青云城 | ⭐ | 开篇 |
| 1/3 | 外门执事羞辱 | 林凡,王虎 | 外门演武场 | ⭐⭐ | 励机 |
| 1/5 | 晨遇神秘老者 | 林凡 | 后山竹林 | ⭐⭐⭐ | 伏笔 |

图例:
- ⭐ 一般事件
- ⭐⭐ 重要事件
- ⭐⭐⭐ 关键转折点

## 时间线说明
- [描述整体时间跨度]
- [说明时间计算规则]
- [特殊时间现象（如：修为流速变化）]
```

## 常见问题

**Q: 时间线太复杂，怎么管理？**

A:
- 使用表格方式呈现（如上模板）
- 只记录主线和重要支线，次要事件可在章节大纲中
- 用"事件 ID"的方式引用，避免重复描述

**Q: 多线并行怎么处理？**

A:
- 每条线单独记录时间线
- 在交汇事件处标注"参见 XX 线"
- 定期检查多条线在同一时间点的位置是否合理

**Q: 修为小说时间流速怎么设定？**

A:
- 明确"修为时间"与"凡间时间"的关系
- 考虑不同层次的流速差异
- 可以用"闭关修炼"来解释时间跳跃
- 在 `docs/world/time-flow.md` 中记录规则

**Q: 时间跳跃怎么处理？**

A:
```
# N 年后

【主角变化】
- 年龄、力力、地位的变化

【世界变化】
- 势力格局的变化
- 重要地点的变化

【未了悬念】
- 之前埋下的伏笔如何收线
- 新的角色关系
```
