# 世界观构建助手

帮助你构建自洽、丰富的小说世界观。

## 构建原则

1. **内部一致性** - 设定之间不能矛盾
2. **服务剧情** - 世界观要为故事服务，不是独立的
3. **适度复杂** - 复杂不是深度，可读性更重要

## 构建维度

### 🌍 力量体系

定义世界中的"力量"来源和规则：

```
【力量来源】
- 灵气/法力来源
- 修炼/获取方式
- 境界限制和瓶颈

【等级划分】
- 每个等级的名称和能力范围
- 突破方式
- 每个等级的大致人数

【特殊能力】
- 是否有特殊天赋/异能
- 获得方式和概率
- 限制和代价
```

### 🗺️ 地理环境

```
【世界规模】
- 完整地图大小
- 主要区域划分

【重要地点】
- 名称、位置、特点
- 相关势力/角色
- 在剧情中的作用

【距离关系】
- 主要地点间的距离
- 正常 travel time
- 特殊交通方式
```

### 🏰 势力组织

```
【组织结构】
- 名称、规模、总部
- 内部等级和分工

【势力关系】
- 与其他势力的关系（联盟/敌对/中立）
- 历史和当前动态

【重要成员】
- 领袖人物
- 核心战力
- 性格特点
```

### 📜 历史背景

```
【世界起源】
- 起源传说（可以是传说，不一定是真相）

【重大事件】
- 改变世界走向的历史事件
- 时间点
- 对当前的影响

【当前时代】
- 大的政治/经济/文化背景
- 主要矛盾
```

## 一致性检查

当添加新设定时，检查：

1. **是否与已有设定冲突？**
   - 力量体系是否一致
   - 地理关系是否合理
   - 势力关系是否符合已有描述

2. **是否影响已发布章节？**
   - 如果是，需要警告用户
   - 列出需要修改的章节范围

3. **是否有逻辑漏洞？**
   - 力量体系是否自洽
   - 势力关系是否有合理的动机

## 常见问题

**Q: 我的力量体系太复杂了，读者会看不懂吗？**

A: 复杂不是深度。重要的是：
- 信息逐步释放，不要一次性全部抛出
- 根据角色视角来介绍，不是说明书
- 只需要读者理解的部分，不需要设定100%完整

**Q: 势力太多记不住怎么办？**

A:
- `docs/world/factions.md` 中记录势力关系图
- 用表格/图示方式呈现
- 只记录与主线相关的势力

**Q: 地点距离怎么计算才合理？**

A:
- 参考现实世界的 travel time（步行 5km/h，马匹 30km/h）
- 考虑是否有修为/超速能力
- 询问自己：读者对这个距离的感知是否合理？
